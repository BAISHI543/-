# 短视频动漫分镜+关键帧一体化（内部工作流工具）

这是一套**内部产线工具**的设计文档集：把「故事/脚本」自动结构化为**分镜表 + 关键帧图**，并沉淀**角色/场景/道具资产**以保证一致性，最终导出给剪辑/动画/配音等下游。

> 目标：你来做动漫；工程师按本文档实现工具；分镜/关键帧产物可直接进入制作。

## 目录结构

- `docs/PRD.md`：产品需求文档（目标/范围/验收）
- `docs/WORKFLOW.md`：端到端流程 + 角色分工（RACI）
- `docs/DATA_MODEL.json`：数据模型（JSON Schema风格）
- `docs/PROMPTS.md`：提示词模板（Bible→ShotList→Keyframe）
- `docs/UI_SPEC.md`：界面与交互细节（三大页面）
- `docs/FILE_NAMING.md`：图片/导出包命名规范

## 快速开始（工程实现建议）

1. 先按 `DATA_MODEL.json` 建表/建接口（Project / Asset / Shot / Keyframe / Revision）。
2. 按 `UI_SPEC.md` 实现三个核心页面：
   - Project/Bible（资产与一致性）
   - Storyboard（分镜编辑 + 关键帧生成/回写）
   - Export（打包导出）
3. 接入生成能力（你们的模型/服务）：先实现「只出结构」再实现「出图」。
4. 以 `PROMPTS.md` 的三段式链路做最小可用（MVP）：
   - Bible 生成/补全
   - ShotList 生成（严格结构化）
   - Keyframe 单镜生成（可重试、可锁定）
5. 用 `PRD.md` 的验收标准做联调与回归。

## 术语（Glossary）

- **Bible（创作圣经）**：角色/场景/道具的“不可漂移”设定集合（带ID，可复用）。
- **Shot（镜头）**：分镜最小单位（含镜头语言、动作、字幕、时长、资产引用）。
- **Keyframe（关键帧）**：每个Shot的一张或多张代表性画面，用于指导后续动画/视频生成。
- **一致性锚点**：用于保证多镜头保持同一角色/场景风格的约束（ID引用 + 视觉锚点描述 + 参考图）。
- **锁定（Lock）**：锁住某些Shot/Keyframe不再被批量重生成或被全局修改影响。

## 约束与默认值

- 目标内容：短视频动漫 30–180 秒，默认 9:16。
- 产物：分镜表 + 关键帧图一体化；强调内部协作与审核节点。
