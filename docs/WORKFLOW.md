# WORKFLOW｜端到端流程与角色分工（RACI）

## 1. 端到端流程（30–180s短视频，默认9:16）

### 里程碑 M0：项目立项（Project Setup）
- 输入：题材、目标平台、时长、画风模板
- 产出：Project创建、默认镜头数范围、风格模板绑定
- 审核点：导演确认方向与约束

### 里程碑 M1：Bible（创作圣经）建立
- 动作：
  - 角色卡：主角/配角，外观锚点、服装、表情组
  - 场景卡：主场景3–8个，光照/色调规则
  - 道具卡：高频关键道具
- 产出：Bible资产（带ID）
- 审核点：美术设定/导演确认“禁改项”

### 里程碑 M2：分镜表（ShotList）生成与定稿
- 动作：从梗概/脚本生成ShotList；人工微调镜头语言与时长
- 约束：前3秒Hook；总时长≈目标时长；镜头数在范围内
- 产出：ShotList（状态：review）
- 审核点：导演锁定（locked）

### 里程碑 M3：关键帧（Keyframe）草图批量生成
- 动作：按shot逐镜生成关键帧（草图/低成本模式），默认每镜 **3+1**：`start/middle/end` + `main`
- 产出：每镜关键帧组（4张）+ revision记录
- 审核点：分镜师/导演快速筛选：通过/需要重生/需要改镜头

### 里程碑 M4：关键帧定稿与一致性校验
- 动作：
  - 对通过镜头锁定
  - 失败镜头“单镜重生”或“替换镜头语言”
  - 同场景镜头批量统一光色（可选）
- 产出：关键帧定稿（final）
- 审核点：导演终审

### 里程碑 M5：导出交付包（Export Package）
- 动作：导出CSV/Excel + 图片包 + JSON +（可选）配音/字幕稿
- 产出：可被剪辑/动画/配音链路直接使用的包
- 审核点：后期确认字段对齐与命名规范

## 2. 角色分工（RACI）

| 阶段 | 任务 | Director(导演/编导) | Storyboard(分镜) | Art(美术设定) | Editor(后期/剪辑) | Admin |
|---|---|---|---|---|---|---|
| M0 | 创建项目/约束 | A/R | C | C | C | R |
| M1 | 角色/场景/道具Bible | A | C | R | I | C |
| M2 | ShotList生成与调整 | A/R | R | C | C | I |
| M3 | 草图关键帧生成 | C | R | R | I | I |
| M4 | 定稿关键帧与一致性 | A | R | R | C | I |
| M5 | 导出交付 | A | C | C | R | I |

说明：R=Responsible执行，A=Accountable最终负责，C=Consulted咨询，I=Informed知会。

---

## 3. 审核节点（建议）
- Gate 1（M1后）：Bible禁改项确认（否则后续必漂移）
- Gate 2（M2后）：ShotList锁定（否则关键帧批量生成会浪费）
- Gate 3（M4后）：关键帧final锁定（作为动画/视频生成的输入基准）

## 4. 常见返工原因与对策
- 角色漂移：缺少角色ID/锚点 → 强制引用角色ID + 参考图
- 节奏拖沓：镜头过多/每镜过长 → 一键压缩镜头数 + 保留节拍点
- 画面信息不清：构图/景别错误 → 提供镜头语言快捷模板（特写/反应/建立镜头）
