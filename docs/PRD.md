# PRD｜短视频动漫：分镜 + 关键帧一体化（内部工作流工具）

## 1. 背景与问题
内部短视频动漫制作的核心痛点：
- 文本到分镜依赖人工，速度慢、质量不稳定。
- 多镜头角色/场景容易“漂移”，返工成本高。
- 分镜、美术、后期各自为政，镜头信息与素材无法对齐。
- 关键帧与分镜表分离，难以追溯“这张图对应哪个镜头/版本/提示词”。

## 2. 目标（Goals）
- 将「梗概/大纲/脚本」快速生成**可编辑的分镜表**（ShotList）。
- 为每个镜头生成**关键帧图**（默认每镜 **3+1**：`start/middle/end` + `main`；可配置数量）。
- 建立可复用的**Bible资产库**（角色/场景/道具），通过ID引用实现一致性。
- 支持内部协作：审核、锁定、版本、回滚、变更记录。
- 导出可交付：Excel/CSV + 图片包 + JSON，供剪辑/动画/配音/字幕链路使用。

## 3. 非目标（Non-goals）
- 不做完整非线编剪辑器（只导出时间轴数据/标注）。
- 不做最终成片渲染/合成（可对接外部工具）。
- 不承诺“一键出最终画面”，本工具以“可控、可复现、可协作”为核心。

## 4. 目标用户与角色
- 编导/导演：定义故事节奏、镜头语言、审核分镜。
- 分镜师：调整镜头、构图、动作分解。
- 美术/设定：维护角色/场景/道具资产与一致性规则。
- 后期/剪辑：使用导出包对齐字幕/配音/节奏。
- 管理者：查看产能、版本、返工原因。

## 5. 典型用户旅程（User Journey）
1) 创建项目 → 填写时长/比例/风格模板 → 导入脚本
2) 生成/补全 Bible（角色/场景/道具）→ 人工确认“禁改项”
3) 生成 ShotList（镜头数、总时长受约束）→ 编导审核通过
4) 批量生成关键帧缩略图（草图模式）→ 分镜师逐镜微调
5) 锁定通过的镜头 → 对失败镜头单独重生/替换镜头语言
6) 输出定稿关键帧（高质量模式）→ 导出交付包

## 6. 核心功能（MVP → V1 → V2）

### 6.1 MVP（两周～一个月可落地）
- Project创建/管理：时长、比例（默认9:16）、目标镜头数范围。
- Bible模块：角色/场景/道具条目CRUD；支持上传参考图；“禁改项”字段。
- ShotList生成：从文本生成结构化镜头列表；支持人工编辑；镜头时长合计约束。
- Keyframe生成：逐镜生成关键帧（每镜1张）；失败可重试；回写到shot。
- 锁定/状态：draft → review → locked → final。
- 导出：
  - 分镜表（Excel/CSV）
  - 图片包（按命名规范）
  - JSON（用于自动化对接）

### 6.2 V1（提升一致性与效率）
- 一致性策略：
  - 角色/场景/道具ID强制引用
  - 角色外观锚点字段 + 参考图优先级
  - 同场景镜头批量统一光色/色调
- 批量操作：批量改景别/运镜、批量加反应镜头、批量重生失败镜头。
- 版本管理：项目版本、镜头版本、关键帧版本；差异对比与回滚。
- 审核流：分镜审核、关键帧审核（可选双人复核）。

### 6.3 V2（生产化协作与下游对接）
- 多人实时协作（锁定冲突处理）、评论与@。
- 输出剪辑时间轴（可选：FCPXML/EDL/简化JSON时间轴）。
- 数据看板：返工率、失败原因、镜头耗时、模型成本。

## 7. 权限与协作（RBAC建议）
- Admin：项目、权限、模板、全局资产管理。
- Director：审核/锁定分镜与关键帧；可导出。
- Artist：编辑Bible、生成/重生关键帧。
- Editor：查看/导出交付包（只读分镜编辑）。

## 8. 版本管理与追溯
- 任意一次生成/重生都产生 `revision`：记录输入、提示词、seed、参考图ID、时间、操作者。
- 支持“锁定”：锁定的Shot不可被批量改写；仅允许显式解锁。

## 9. 导出与交付
- Excel/CSV：按shot_id排序，包含时长、字幕、音效、资产引用。
- 图片包：关键帧PNG/JPG。
- JSON：完整数据（用于下游自动化）。

## 10. 约束与质量要求
- 短视频节奏：前3秒必须有Hook镜头（强信息/强情绪/强冲突）。
- 时长约束：镜头时长总和≈目标时长（允许±5%）。
- 一致性：同一角色ID在不同镜头外观不可漂移（发型、瞳色、标志物必须一致）。

## 11. 验收标准（Acceptance Criteria）
- 给定一段100–600字脚本，能在一次操作内生成结构化ShotList（含时长、字幕、镜头语言、资产引用）。
- 对ShotList可逐镜编辑并保存；锁定后不会被批量操作覆盖。
- 对任意shot可生成关键帧并与shot绑定；可重试并保留历史revision。
- 可导出：
  - 分镜CSV/Excel（字段完整）
  - 图片包（命名正确、可追溯shot_id）
  - JSON（符合 `DATA_MODEL.json`）
- **强制导出校验**：存在 `final/locked` 镜头未选择 `main` 关键帧时，系统必须禁止导出并给出缺失清单。
